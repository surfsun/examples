RabbitMQ 消息模式教学

8种经典消息模式 · 交互式演示 · 知识问答考核

🟢 基础模式

简单队列模式

一个生产者 → 一个队列 → 一个消费者，最基本的消息传递。

Simple Queue

工作队列模式

多个消费者竞争消费同一队列，实现负载均衡和并行处理。

Work Queue

发布/订阅模式

一条消息广播到所有订阅者，适用于系统通知和新闻推送。

Fanout Exchange
🟡 进阶模式

路由模式

根据 routing key 精确匹配，将消息路由到指定队列。

Direct Exchange

主题模式

使用通配符（* 和 #）匹配 routing key，灵活的消息路由。

Topic Exchange
🔴 高级模式

死信队列

消息处理失败或过期后的兜底方案，确保消息不会丢失。

Dead Letter Exchange

延迟队列 (TTL)

基于 TTL + DLX 实现的延迟消息，适用于订单超时等场景。

TTL + DLX

插件延迟队列

基于 x-delayed-message 插件，精准延迟，无队头阻塞。

Plugin Delay
